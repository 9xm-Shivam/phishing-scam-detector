# 🛡️ PhishGuard AI — Phishing & Scam Detector

![Python](https://img.shields.io/badge/Python-3.10+-blue?logo=python&logoColor=white)
![Flask](https://img.shields.io/badge/Flask-3.0-black?logo=flask)
![scikit-learn](https://img.shields.io/badge/scikit--learn-1.4-orange?logo=scikitlearn)

> An AI-powered web application that detects **phishing URLs** and **scam emails/messages** in real time using machine learning models trained on cybersecurity datasets.

---

## ✨ Features

| Feature | Details |
|---|---|
| 🔗 **URL Phishing Detector** | Analyses 12 engineered features — entropy, TLD risk, IP usage, subdomains, HTTPS, and more |
| 📧 **Email / Text Scam Scanner** | TF-IDF + Logistic Regression on 5000 bi-gram features to detect scam language patterns |
| 📊 **Risk Score** | 0–100 risk score with Safe / Suspicious / Phishing verdict |
| ⚡ **Heuristic Fallback** | Works without ML models via rule-based engine (no training required to demo) |
| 🌐 **REST API** | JSON endpoints for integration with other tools |

## ⚙️ Setup & Installation

### 1. Clone the repository
```bash
git clone https://github.com/9xm-shivam/phishing-scam-detector.git
cd phishing-scam-detector
```

### 2. Create a virtual environment
```bash
python -m venv venv

# On Windows:
venv\Scripts\activate

# On macOS/Linux:
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Train the ML models *(optional but recommended)*
```bash
python train_model.py
```

> **Without training:** The app uses a heuristic rule-based engine automatically.
> **With training:** Full ML predictions with confidence scores.

Training output example:

### 5. Run the app

## 🧠 How It Works

### URL Feature Engineering (12 features)

| Feature | Why It Matters |
|---|---|
| `has_ip` | Phishing URLs often use raw IPs to avoid domain detection |
| `has_at_symbol` | `@` before domain hides the real destination |
| `has_https` | Lack of HTTPS is a red flag |
| `tld_risk` | Free TLDs (.tk, .ml, .ga) are overwhelmingly abused |
| `entropy` | Obfuscated/random domains have high Shannon entropy |
| `subdomain_count` | Multiple subdomains used to mimic trusted domains |
| `suspicious_keywords` | Presence of: login, verify, secure, paypal, etc. |

### Text Classification Pipeline

```
Raw Text → TF-IDF Vectorizer (5000 features, bigrams)
         → Logistic Regression Classifier
         → Phishing probability score
```
